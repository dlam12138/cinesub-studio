# v0.7.1 法语 OCR 能力复核

## 用户目标

继续执行 v0.7.1 真实媒体验收，在冻结环境前安装并验证法语 Windows OCR 能力。

## 已知事实与证据

- Windows 设置已显示“法语（法国）”。
- `Get-WinUserLanguageList` 已包含 `fr-FR`，但当前用户仅安装了中文 Language Experience Pack。
- `OcrEngine.AvailableRecognizerLanguages` 实际枚举仍只包含 `en-US`、`zh-Hans-CN` 和 `zh-Hant-HK`，不包含 `fr-FR`。
- Windows Update、BITS 和 Delivery Optimization 服务处于运行状态。
- 系统提供 `Install-Language`，但 agent 的当前执行策略阻止了管理员提权调用；此前 `Add-WindowsCapability` 也因 Windows Update `0x80D03805` 下载错误失败。

## 本轮决策

- 不把语言列表中出现法语误判为 OCR 安装成功。
- 不用英语 OCR 引擎替代法语引擎生成验收证据。
- 在 `fr-FR` 真正出现于 OCR 可用语言枚举前，不冻结环境、不进行六视频预筛和正式矩阵。

## 实际操作

- 复核了用户语言列表、Language Experience Pack 和 Windows OCR 实际可用语言。
- 检查了 `Install-Language` 命令及 Windows Update 相关服务状态。
- 尝试以管理员权限调用 `Install-Language`，但在启动前被当前执行策略阻止，没有修改系统。
- 重新打开 Windows“语言和区域”设置页，请用户在法语的“语言选项”中手动安装可用语言功能。

## 验证结果

- `fr-FR` 用户语言：已添加。
- `fr-FR` Windows OCR 引擎：仍不可用。
- 本轮未更改产品代码、Profile 解析或验收阈值。

## 未解决问题与下一步

- 用户需在 Windows 设置中完成法语语言功能安装。
- 安装后立即重新枚举 `OcrEngine.AvailableRecognizerLanguages`，并用 OCR bridge 创建法语和简体中文引擎。
- OCR 验证通过后，执行完整回归，创建并推送 v0.7.1 preparation 提交，再冻结环境进入预筛。
