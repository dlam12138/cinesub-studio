缺失组件处理

模型或 FFmpeg 缺失时：
  • FFmpeg：放入 tools/ffmpeg/，或配置 CINESUB_FFMPEG / FFMPEG_PATH。
  • ASR 模型：放入 models/，或使用项目内 Hugging Face cache。
  • 单个处理的高级识别参数里可以选择模型下载源，也可以启用“只使用本地已下载模型”。这里仅提供处理指引，不执行网络动作。

语音识别与翻译

使用流程：选择视频 → 配置识别参数 → 选择翻译设置 → 点击"开始识别"。
第一次使用某个模型会自动下载；国内网络可尝试 hf-mirror.com。翻译接口的 API Key 不会持久化到磁盘或日志中。

步骤 1 · 选择视频

大电影不要上传，直接填本机路径。后台读取原片并抽音频到 work/，原片不会被复制。

适合蓝光级别大文件。原片保持在原位置，抽取的音频写入项目 work/，字幕和质检结果写入 output/。
预检只看路径、大小、扩展名和预计输出名，不读取视频内容。
上传小样本测试 上传会先保存一份到 uploads/；大电影请使用上面的本机路径。

步骤 2 · 语音识别

先选择视频的语言情况；不了解时使用默认的“自动检测”。

识别方式默认自动检测

识别置信度只会出现在日志和复核提示中，不会自动换模型或重跑。

识别性能

模型越大越慢、越吃内存。small 适合 CPU；medium / large-v3 更准；distil-large-v3 更偏 GPU。
CPU 最兼容；CUDA 速度更快但要求 NVIDIA 显卡环境；自动适合你不确定机器配置时。
高级识别参数
CPU 通常选 int8，省内存；CUDA 通常选 float16；float32 最吃内存，只在你明确需要时用。
首次使用模型会下载到项目目录。国内网络可用镜像；模型下好后可勾选“只使用本地模型”。
识别时同时尝试几条候选文本。默认 5 比较稳；越大越慢，不等于一定更准。
自动跳过无声片段，电影对白建议开启；极短音频或音乐素材异常时再关闭。 离线或模型已下载时启用；第一次跑某个模型不要勾，否则会找不到模型。 让 Whisper 记住上一段内容。长对白建议开启；歌词、跳切、多语言混杂可关闭。

步骤 3 · 翻译设置

语言配置管识别习惯、翻译风格和质检阈值；模型接口只管 API 和大模型名称。

不懂就用当前默认语言配置。它会自动带入识别方式、目标语言、既有翻译风格和质检阈值。
勾选后会调用 Provider 里的 API。只想生成原文 SRT 时不要勾。

字幕目标语言。中文用户默认 zh-CN；繁体字幕选 zh-TW。
双语适合校对和学习；纯翻译适合最终观看版。
高质量模式需要配置 Pro 模型并严格保留字幕 ID 和时间轴；WenYi 单独模式已有已知安全回归，仅供实验比较。
达到该静音间隔时开始新场景；过大场景会优先在最长字幕间隔处分批。
每批翻译多少条字幕 每次丢给大模型多少条字幕。越大请求越少，但失败后重试成本更高。
0=保守，1=创意 控制翻译发挥程度。字幕要稳，默认 0.2；越高越自由，也越可能跑偏。
高级翻译参数
让模型参考相邻字幕，人物代词和剧情衔接更稳。极短字幕测试时可关闭。
翻译时参考前后多少条字幕 默认参考前后 3 条。数值越大越连贯，但 prompt 更长、API 消耗更高。
