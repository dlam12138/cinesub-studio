# `wenyi` 翻译质量借鉴设计

## 来源与许可

本设计审阅基准为 [BigDawnGhost/wenyi commit `d3e12d6`](https://github.com/BigDawnGhost/wenyi/commit/d3e12d65080750d960ec2ff4f7ef9ab9d233b793)。该项目使用 MIT License；若后续复制或改写具有实质性的代码或提示词，必须在对应源码和第三方声明中保留原版权与许可证归因。

本轮只记录设计，不复制 `wenyi` 源码、不安装其依赖、不调用付费 LLM。实施归入阶段七“翻译质量与恢复”，不并入阶段三 ASR 评测。

## 适配顺序

1. 在现有批次翻译中加入最近译文滚动上下文；继续要求输入/输出 ID 严格一一对应，结构重试仍失败时逐条兜底。
2. 加入无需模型的译文/原文长度比异常检查，并继续使用 Language Profile 人工术语表。
3. 增加原文—译文语义审校，检查漏译、增译、误译、人称和术语；默认只生成报告与人工复核条目。
4. 只有显式启用时才根据审校意见定向重译或执行中文润色；条数、ID、结构校验失败或 API 调用失败时保留原译文。

## 明确不引入

- EPUB/FB2/TXT 解析与回填。
- 小说章节、全书预扫或全书梗概。
- SQLite 术语库和自动术语裁定。
- DeepSeek strong/cheap/fast 三档 Provider 模型。
- 小说专用标点、文学段落切分和长篇章节编排。

所有阶段继续使用当前 Provider 的 API Key、API Base 和单一 LLM 模型；Provider schema 不承载翻译风格或字幕状态。输出必须保持原 SRT 的 ID、条数和时间轴。

## 兼容与成本保护

- 后续为 `llm_stages` 增加总开关 `enabled`，默认 `false`；旧 Profile 缺少该字段时同样视为关闭。
- 现有内置 Profile 虽已包含 `proofread_source`、`polish_translation`，但在 `enabled` 未显式为 `true` 时不得触发任何额外 API 请求。
- 自动修复默认关闭，语义审校默认只读；真实 API E2E 必须显式授权。
- 翻译缓存键必须覆盖有效提示词、术语、滚动上下文、阶段设置和源字幕内容，避免错误复用。
- 日志、异常和报告不得包含 API Key；审校失败、润色失败或结构不合法均降级为保留原译文。

## 启用前 A/B 验收

使用固定字幕集分别运行当前流程与增强流程，人工盲评漏译、误译、术语一致性、角色口吻、自然度和字幕长度，同时记录请求次数、token/费用（Provider 可返回时）与耗时。只有质量有稳定改善、结构无回归且成本可接受时，才允许把对应阶段作为用户可选能力发布；默认仍保持关闭。
