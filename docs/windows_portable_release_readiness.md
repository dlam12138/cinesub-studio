# Windows Portable Release Readiness

M14 defines a repeatable Windows portable staging workflow. It is release-readiness work, not an official release.

M14 does not create an installer, Electron shell, Tauri shell, auto-updater, code signing flow, model hub, model downloader, FFmpeg downloader, or committed release artifact.

## Portable Staging Layout

The intended staging folder is:

```text
CineSubStudio/
  start_web.ps1
  start_app.py
  README.md
  README_QUICKSTART.md
  requirements.txt
  pyproject.toml
  install.ps1
  src/
  web/
  config/
  docs/
  tools/
    ffmpeg/
      README_PLACE_FFMPEG_HERE.txt
  models/
    README_PLACE_MODELS_HERE.txt
  runtime/
  output/
  logs/
  work/
  uploads/
```

The staging folder is generated locally and must remain untracked.

## Included Files

`scripts/build_portable.ps1` uses tracked files as the source inventory, then applies a safe allowlist before copying. The allowlist includes:

- `start_web.ps1`
- `start_app.py`
- `install.ps1`
- `README.md`
- `requirements.txt`
- `pyproject.toml`
- `src/`
- `web/`
- `docs/`
- `config/*.example` and `config/*.sample`

The helper is intentionally not a whole-repository copier.

## Excluded Files

The staging helper excludes development, runtime, secret, and generated files, including:

- `.git/`, `.venv/`, `.cache/`, `.tmp/`
- `tests/`, `acceptance/`, `audit/`, `reports/`
- `input/`, `output/`, `work/`, `uploads/`, `archive/`, `failed/`, `logs/`
- `models/*`
- `tools/python/`, `tools/wheelhouse/`, `tools/cuda/`, `tools/ffmpeg/`
- `config/providers.local.json`
- `config/language_profiles.local.json`
- `.env` and `.env.*`
- files whose names indicate tokens, secrets, passwords, or API keys
- media, subtitle, log, temp, zip, and runtime state artifacts
- `project_evaluation_report.md`

These exclusions are enforced even if a local secret file is accidentally tracked.

## FFmpeg, Models, Config, Runtime

FFmpeg binaries belong under:

```text
tools/ffmpeg/bin/
```

They are not committed and are not generated by M14. A user may also set `CINESUB_FFMPEG` or `FFMPEG_PATH`.

ASR model files belong under `models/` or the supported project-local Hugging Face cache. M14 does not download models and does not add a model hub.

Local Provider and Language Profile config is written under `config/` at runtime. API keys must never be staged from source.

Runtime files stay inside `runtime/`, `output/`, `logs/`, `work/`, and `uploads/`.

## Commands

Create a dry-run manifest without writing files:

```powershell
.\scripts\build_portable.ps1 -DryRun -StagingDir dist\CineSubStudio-portable
```

Create a local staging folder:

```powershell
.\scripts\build_portable.ps1 -StagingDir dist\CineSubStudio-portable
```

Create an optional local zip:

```powershell
.\scripts\build_portable.ps1 -StagingDir dist\CineSubStudio-portable -Zip
```

Launch from the staged folder:

```powershell
.\start_web.ps1
```

Run non-interactive validation:

```powershell
.\start_web.ps1 -Smoke -NoBrowser -NonInteractive
```

The default local URL is:

```text
http://127.0.0.1:7860/
```

## Relationship To The Older Release Builder

`scripts/build_portable_release.py` is preserved unchanged. It targets an older portable-runtime release-candidate shape that can include a copied portable Python runtime.

`scripts/build_portable.ps1` is the M14 Windows staging helper. It prepares a conservative source-based staging folder and documents what belongs in a future portable package. It does not replace the older builder.

## Artifact Policy

Generated staging folders and zips must stay under ignored paths such as `dist/` or `release/`. They must not be committed, pushed, tagged, or attached as official release artifacts.

If a local zip is generated during validation, acceptance evidence must record the path, whether it remained untracked, and that it is not an official release.

## Known Limitations

M14 does not verify offline standalone operation. Python and `.venv` are not bundled by this staging workflow.

Smoke validation does not run media processing, ASR model loading, translation requests, FFmpeg download, model download, muxing, dubbing, TTS, or lip-sync.
